agents have no long-term memory. every session starts from zero. lessons vanish. mistakes repeat. knowledge never compounds.

shellbrain fixes this. it gives agents durable, structured memory grounded in what actually happened—and retrieves it the moment a problem resembles something already seen.

the result. agents that solve faster, fail less, and recall things they would never think to look for.

procedural memory

every problem you have ever solved.

shellbrain remembers every problem, every working solution, and every failed attempt.

failed tactics matter. knowing what didn't work is half the solution.

every solution and every failure is linked to the problem it belongs to. without that link, the memory is invalid. shellbrain enforces this.

a problem, its solutions, and its failed tactics form a scenario. shellbrain remembers all of your scenarios.

the next time a similar problem appears, shellbrain retrieves the relevant scenarios and gives the agent a head start.

semantic memory

knowledge that never silently mutates.

shellbrain remembers facts about the world, your repo, and your code.

shellbrain never overwrites facts. when a fact changes, it stores the change that invalidated it, stores the updated fact, and links all three: old fact → change → new fact.

history is always preserved. you can trace any belief back through every revision that shaped it.

shellbrain also remembers how you like to work—repo conventions, style decisions, what should win when there is ambiguity. these are durable memories, learned from working with you.

episodic memory

nothing is invented.

shellbrain records every interaction between you and your agent. all dialog. all tool use. every event from every work session.

this is episodic memory. it is the foundation everything else is built on.

every durable memory must trace back to a real event. evidence is never fabricated. if the evidence is ambiguous, shellbrain skips the write entirely.

shellbrain would rather have no memory than a memory it cannot justify.

associative memory

recall that looks like creativity.

memories relate to each other. shellbrain captures these relationships in two ways.

explicit associations are formal links in a knowledge graph—structural relationships stored as first-class records.

implicit associations are computed from semantic similarity via vector embeddings.

when a problem triggers retrieval, shellbrain doesn't just return direct matches. it seeds candidates from semantic and keyword search, then expands outward—through linked scenarios, fact-update chains, association edges, and semantic neighbors. it scores, deduplicates, and returns a bounded context pack.

every result explains why it was included. the pack is capped. retrieval is precise, not exhaustive.

chaining through multiple associative hops surfaces memories the agent would never think to look for. recall starts to look like creativity.

feedback

utility is earned, not assumed.

a memory can be critical for one class of problem and useless for another.

shellbrain does not assign a single global score. it records utility per problem—a vote for each memory against each problem it was recalled for.

global utility is derived from these observations. never assumed.

four operations. that's it.

read

retrieve memories related to a concrete problem. does not mutate state.

events

inspect recent evidence from the active session. how the agent knows what just happened.

create

add one durable memory. at least one evidence reference is required.

update

record utility, evolve facts, add associations, or archive.
