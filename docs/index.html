docs

agents have no long-term memory. every session starts from zero. lessons vanish. mistakes repeat. knowledge never compounds.

shellbrain gives agents structured, long-term memory—typed, evidence-grounded, and retrieved the moment a problem resembles something already seen.

use model

shellbrain is a skill. your agent activates it automatically at the start of every conversation. you never manage memory directly.

compatibility

works with claude code, cursor, and codex. installs as a native skill in each. memory is shared across all of them.

storage

typed memory in a knowledge graph. scenarios, facts, preferences, associations—distinct node types with structural edges. not markdown files.

retrieval

BM25 keyword search fused with semantic embeddings. candidates expand through graph edges. bounded, scored, and every result explains why it was included.

procedural memory

every problem you've ever solved.

every problem, every working solution, and every failed attempt—linked together as a scenario. without that link, the memory is invalid. shellbrain enforces this.

the next time a similar problem appears, the agent already knows what to try and what to skip.

this is case-based reasoning. problems are never stored in isolation—only as structured cases with outcomes.

semantic memory

knowledge that never silently mutates.

facts are never overwritten. when knowledge changes, shellbrain stores the old fact, the change that invalidated it, and the new fact—linked as an immutable revision chain.

shellbrain also remembers how you like to work—repo conventions, style decisions, what should win when there is ambiguity.

facts, preferences, and changes are distinct typed nodes. each with its own retrieval and revision semantics.

episodic memory

grounded in truth.

every durable memory must trace back to a real event—dialog, tool use, or session activity. evidence is never fabricated. if it is ambiguous, shellbrain skips the write entirely.

shellbrain would rather have no memory than a memory it cannot justify.

three-layer validation—schema, semantic, integrity—rejects anything that can't cite its evidence.

associative memory

recall that looks like creativity.

explicit associations are formal edges in the knowledge graph. implicit associations are computed from vector similarity. both expand recall outward—through linked scenarios, revision chains, and semantic neighbors.

multi-hop traversal surfaces memories the agent would never think to look for. recall starts to look like creativity.

feedback

utility is earned, not assumed.

after retrieval, agents vote on which memories actually helped. utility is recorded per problem, not globally. the graph sharpens itself over time.

four operations. that's it.

read

retrieve memories for a concrete problem. does not mutate state.

events

sync the session transcript. returns episode event IDs the agent can cite as evidence.

create

write one durable memory. at least one evidence reference is required.

update

record utility, evolve facts, add associations, or archive.
